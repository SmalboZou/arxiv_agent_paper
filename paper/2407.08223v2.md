1. 论文基本信息

- **标题**：SPECULATIVE RAG: ENHANCING RETRIEVAL AUGMENTED GENERATION THROUGH DRAFTING 
- **作者**：Zilong Wang, Zifeng Wang 等 （主要来自加州大学圣地亚哥分校、Google Cloud AI Research 以及 Google DeepMind）
- **会议**：录用于 ICLR 2025 

## 2. 研究背景与动机

虽然大型语言模型（LLMs）在问答任务中表现出色，但在处理需要最新信息或冷门事实的知识密集型问题时，容易产生事实错误和内容幻觉 。检索增强生成（RAG）通过引入外部知识库有效缓解了这一问题 。然而，现有 RAG 技术面临以下挑战：

- **上下文过长与效率低下**：为了包含所有必要信息，RAG 通常会检索多个文档，这不可避免地增加了输入 LLM 的长度。超长上下文不仅增加了推理延迟，也容易使模型产生“迷失在中间（lost-in-the-middle）”的位置偏差 。
- **现有改进方案的局限性**：例如 Self-Reflective RAG 需要对通用大模型进行特定的指令微调以生成反思标签，这需要耗费大量资源 ；而 Corrective RAG (CRAG) 虽然使用了外部评估器，却缺乏高层级的推理能力 。现有的工作往往侧重于提高上下文质量，而忽略了延迟问题 。

## 3. 核心创新点

为了在效率与性能之间取得平衡，本文提出了一种名为 **SPECULATIVE RAG (推测性 RAG)** 的框架，其核心创新点包括：

1. **分而治之的起草与验证架构**：系统将任务卸载并分解，利用一个较小的“专家起草器（Specialist RAG Drafter）”并行生成多个 RAG 候选答案（草稿），然后由一个较大的“通用验证器（Generalist LM）”对这些草稿进行高效验证 。
2. **多视角子集降低输入负担**：每个草稿都是基于检索文档的某个不同子集生成的。这不仅提供了关于证据的多种视角，还有效减少了每个草稿的输入 token 数量，从而增强了对文档的理解并缓解了长上下文偏差 。
3. **大型通用模型免微调**：系统中的通用大模型（验证器）只需利用其本身自带的语言建模能力进行验证，不需要进行任何额外的指令微调即可与小模型配合工作 。
4. **显著降低延迟**：将繁重的阅读长文档工作（起草）委派给小模型并行处理，大模型仅对提取好的草稿进行单次无偏的验证，极大提升了生成速度 。

## 4. 架构详解

SPECULATIVE RAG 的执行流程主要包含以下两大模块：

### 4.1 小型专家起草器 (Specialist RAG Drafter)

起草器是一个经过指令微调的较小体积语言模型，它是理解检索文档并生成回答的“专家” 。

- **多视角采样 (Multi-Perspective Sampling)**：面对用户查询，检索系统首先会获取一组文档。为了减少冗余并增加文档子集的多样性，模型使用指令感知嵌入模型和 K-Means 算法对文档的内容相似度进行聚类 。随后，从每个聚类簇中采样一个文档，构成多个包含多元视角的“文档子集” 。
- **并行起草 (RAG Drafting)**：将这些文档子集并行送入起草器模型。起草器会基于每个子集，同时生成“答案草稿（$\alpha$）”以及对应的“支持理由（$\beta$，Rationale）” 。

### 4.2 大型通用验证器 (Generalist RAG Verifier)

验证器是一个现成的通用预训练大语言模型。验证器不再需要去阅读冗长重复的原始检索文档，而是直接对起草器输出的“草稿-理由”对进行验证 。

- **分数计算**：验证器利用自身的语言建模目标来计算各个草稿的置信度分数 ：

  - **自洽性分数 (Self-consistency score)**：计算给定问题下，模型生成该“答案草稿及理由”的条件概率，概率越高代表越符合自洽逻辑 。
  - **自反思分数 (Self-reflection score)**：在提示中加入反思语句（例如：“你认为这个理由支持该答案吗，是或否？”），并计算模型生成肯定回答（"Yes"）的概率 。

- **结果输出**：综合计算得分后，系统选择置信度分数最高的草稿作为最终答案输出 。

## 5. 实验结果与结论

作者在 TriviaQA、MuSiQue、PopQA、PubHealth 和 ARC-Challenge 5个基准数据集上进行了广泛的测试 。

- **卓越的准确率与速度**：SPECULATIVE RAG 实现了当前最优（State-of-the-art）的性能表现，并且延迟更低 。
- **数据表现**：相较于常规 RAG 系统（例如 Mixtral-Instruct8x7B），本框架在 PubHealth 数据集上的准确率提升了最高 12.97%，同时延迟（Latency）下降了 50.83% 。在处理 MuSiQue 等复杂的多跳推理（Multi-hop reasoning）任务时，该框架也展现出极佳的性能 。
